# AGENTS.md

## 基本方針

- ユーザーの指示に従ってください。
- 重大なセキュリティ侵害、秘密情報の漏えい、大規模な削除、意図しない外部公開につながる作業は、実行前に確認してください。
- 説明や報告は日本語で行ってください。

## 作業ルール

- 変更前に関連ファイルを確認してください。
- 変更は小さく、目的に集中してください。
- 既存の実装パターン、命名、フォーマットに合わせてください。
- ファイル入出力は UTF-8 で行ってください。

## 変更後の確認

- 可能な範囲でテスト、ビルド、lint、実行確認を行ってください。
- 実行できなかった確認がある場合は、その理由を報告してください。

## ドキュメント

- ドキュメントは `docs/` 配下に置いてください。
- 仕様や設計は `docs/specification.md`、運用手順は `docs/operations.md` を基本にしてください。
- 仕様変更、画面変更、API変更、データ構造変更、外部連携変更、デプロイ手順変更をした場合は、必ず `docs/specification.md` も同じコミットで更新してください。
- `docs/specification.md` がない場合は作成してください。

## Git

- 変更があった場合は、必要に応じて意味のある単位で `git commit` してください。
- 作業開始時に `git status` を確認し、既存の未コミット変更を不用意に上書きしないでください。
